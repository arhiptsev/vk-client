# vkclient — веб (сборка)

Офлайн-просмотр экспорта VK в браузере. База и медиа обрабатываются **только на вашем компьютере**, на сервер ничего не отправляется.

## Как запустить

1. Откройте файл **`index.html`** в браузере (Chrome или Edge).
   - Можно дважды щёлкнуть по `index.html` в проводнике.
   - Папку `dist` целиком нельзя переименовывать и нельзя удалять каталог **`assets`** — в нём лежит приложение.

2. На стартовом экране выполните два шага (см. ниже).

3. После загрузки базы откроется список диалогов.

## Шаг 1. База данных

Нажмите **«Выбрать .db»** и укажите файл SQLite с экспортом VK, например:

- `sqlite.db`

Это тот же файл, что используется в мобильной версии (`front`) после подготовки базы (`npm run db:prepare`).

## Шаг 2. Медиа (фото, аудио, видео)

В базе хранятся только **имена файлов**. Чтобы показывались вложения, выберите папку с медиа:

1. Нажмите **«Выбрать папку с медиа»**.
2. Укажите **родительскую** папку, внутри которой есть каталоги:
   - `photos/`
   - `audios/`
   - `videos/`

Пример структуры:

```text
моя_папка_с_медиа/
├── photos/
├── audios/
└── videos/
```

В диалоге выбора нужно указать именно **`моя_папка_с_медиа`**, а не отдельно `photos` или `videos`.

Браузер **не** читает путь с диска вроде `C:\Users\…` — только папку, выбранную через диалог. Папку с медиа можно сменить позже в шапке приложения.

### Альтернатива: URL медиа-сервера

Если файлы раздаются по HTTP, вместо папки можно указать **URL медиа-сервера** (на стартовом экране или в шапке). Локальная папка имеет приоритет над URL.

## Содержимое папки `dist`

| Файл / папка      | Назначение                          |
|-------------------|-------------------------------------|
| `index.html`      | Точка входа — откройте в браузере   |
| `assets/index.js` | Приложение (включая SQLite WASM)    |
| `assets/style.css`| Стили                               |
| `README.md`       | Эта инструкция                      |

## Частые вопросы

**Пустые вложения, только подписи с именами файлов**  
Не выбрана папка с медиа или неверная структура каталогов — нужны именно `photos`, `audios`, `videos`.

**Нужен интернет?**  
Нет. После сборки всё работает локально.

## Сборка из исходников (для разработчиков)

```bash
cd web
npm install
npm run build
```

Результат снова появится в `web/dist/`.
